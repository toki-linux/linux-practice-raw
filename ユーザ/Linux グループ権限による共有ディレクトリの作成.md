# Linux グループ権限による共有ディレクトリの作成

## 概要

Ubuntu上で複数ユーザーが同じファイルを共同編集できるように、共有ディレクトリとグループ権限を設定した。

今回の目的は、`root` 権限を使わずに、一般ユーザー同士でファイルを編集・確認できる状態を作ること。

## 構成

- 共有ディレクトリ: `/srv/group-share`
- 共有ファイル: `/srv/group-share/test.txt`
- ユーザー:
  - `toki`
  - `tenn`
- 目的:
  - `toki` と `tenn` の両方が `test.txt` を編集できる
  - 編集内容を互いに確認できる
  - 編集時に `sudo` を使わない

## 実施内容

1. 共有用グループを作成
2. `toki` と `tenn` を共有グループに追加
3. `/srv/group-share` を作成
4. 共有ディレクトリの所有グループを変更
5. グループに読み書き実行権限を付与
6. setgid を設定し、配下に作られるファイルのグループを揃える
7. `test.txt` を作成し、両ユーザーで編集できることを確認

## 確認できたこと

`toki` と `tenn` の両方で、`/srv/group-share/test.txt` を編集できた。

また、どちらのユーザーも `root` 権限を使わずに編集でき、編集後の内容を互いに確認できた。

## 学んだこと

Linuxでは、複数ユーザーでファイルを共有したい場合、毎回 `sudo` を使うのではなく、グループ権限を使ってアクセス権を設計する。

ファイル単体の権限だけでなく、そこに到達するまでのディレクトリ権限も重要になる。

また、共有ディレクトリでは `setgid` を使うことで、配下に作成されるファイルのグループを共有グループに揃えやすくなる。

## 実務での考え方

`/home/ユーザー名` 配下は個人用の領域として扱われることが多いため、複数ユーザーで共有するファイルは `/srv` などの共有用途に適した場所に置く方が自然。

今回のように `/srv/group-share` を使うことで、個人のホームディレクトリに依存せず、共有用の作業場所として管理しやすくなる。
